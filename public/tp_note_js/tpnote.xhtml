TP Noté






Cette partie de l'exercie sert à apprendre comment faire un tableau complexe en HTML avec l’utilisation de la balise caption pour le titre et la description, ce qui améliore l’accessibilité et la sémantique. J’ai appris à appliquer des styles CSS précis sur les différentes parties du tableau, comme les cellules d’en-tête th, les cellules normales td, et l’utilisation de `position: relative` pour mieux contrôler l’agencement des éléments internes. La création d’éléments visuels comme les cercles orange avec un chiffre centré à l’intérieur m’a permis de manipuler le modèle de boîte CSS, les flexbox et les propriétés comme `border-radius` pour arrondir les formes. Ce travail m’a semblé intéressant pour progresser dans la mise en page précise d’éléments graphiques dans un tableau, ce qui est souvent utile pour représenter des données musicales, techniques ou autres. Le niveau de difficulté était modéré : il faut bien comprendre le CSS pour obtenir un rendu propre et harmonieux. Le résultat est clair, fonctionnel et visuellement attrayant, avec une bonne hiérarchie visuelle, ce qui montre que le code est bien structuré et adapté à cet usage.





Ici, on génère du contenu HTML avec JavaScript à partir d’une structure de données JSON, qui est ici un objet représentant les accords de guitare. J’ai appris à manipuler le DOM pour créer un tableau complet avec ses éléments table, caption, body, et à y insérer des lignes et cellules en fonction des valeurs contenues dans l’objet. La gestion conditionnelle d’éléments (création ou non d’un div selon la valeur de la case) m’a aussi permis de mieux comprendre comment afficher uniquement ce qui est nécessaire. Le code fait un bon usage de la modularité avec une fonction `createAccord` réutilisable, ce qui est une bonne pratique en programmation. Le niveau de difficulté était un peu plus élevé que dans les exercices précédents car il faut bien comprendre les boucles imbriquées, la manipulation avancée du DOM, et la structure des données. Cet exercice est très instructif car il montre comment relier des données structurées à un affichage graphique, ce qui est fondamental maintenant. Le résultat obtenu est fidèle à la représentation des accords et le tableau est bien construit ce qui prouve que le code est efficace et bien pensé.






Cette dernière question m’a permis d’approfondir mes connaissances sur la manipulation dynamique du DOM en JavaScript, en intégrant à la fois la génération d’éléments HTML à partir de données structurées et la gestion du temps avec `setInterval` pour créer un effet de dictée musicale. J’ai appris à gérer un cycle d’affichage automatique et aléatoire d’accords, en alternant entre démarrage et arrêt via un bouton, ce qui implique une bonne compréhension des variables globales, des timers, et de la modification dynamique du contenu HTML. Le niveau de difficulté était plus élevé que les exercices et questions précédentes car il combine plusieurs notions : création d’éléments HTML, manipulation d’événements, gestion d’intervalles temporels, et interaction utilisateur. Cet exercice est également très utile pour mon apprentissage (je veux aller en dev web svp) , car il rapproche la programmation événementielle et la gestion d’état dans une interface web, compétences essentielles pour rendre les pages interactives et vivantes. Le rendu fonctionnel est fluide, le code est bien organisé et le mécanisme de dictée marche parfaitement, ce qui témoigne d’une bonne maîtrise des concepts utilisés.
