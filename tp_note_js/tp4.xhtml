TP 4






Ici, on a travaillé sur la création dynamique d’un tableau HTML de 50 lignes par 50 colonnes. Chaque cellule de ce tableau possède une couleur de fond choisie aléatoirement parmi huit couleurs définies dans un objet JavaScript. Le HTML sert à poser une structure simple avec un titre, un conteneur vide pour accueillir le tableau généré et une liste destinée à afficher les fréquences d’apparition de chaque couleur. Le fichier CSS ne sert qu'à enlever les marges internes des cellules et leur donner une taille carrée uniforme. Le JavaScript utilise un objet colorMap qui stocke les associations nom-couleur en hexadécimal, et un script utilise des boucles pour générer les 2 500 cellules du tableau. À chaque rafraîchissement de la page, la couleur est choisie aléatoirement. Ensuite, le code compte combien de fois chaque couleur apparaît et affiche ce résultat sous forme de liste. Cet exercice m’a aidé à comprendre comment créer du contenu avec JS directement dans la page. J’ai mieux compris comment utiliser le DOM et appliquer des styles via le code. Ce n’était pas trop dur, il fallait juste rester organisé.






Dans cet exercice, j’ai travaillé sur l’ajout d’interactions via JavaScript. Le but était de rendre chaque case du tableau interactive, en ajoutant des boutons qui changent les couleurs des cases. Le HTML était toujours structuré proprement, et le CSS servait principalement à donner une apparence claire aux cellules : bordures, taille fixe, et couleurs de fond. En JavaScript, j’ai utilisé un gestionnaire d’événements pour écouter le clic sur chaque bouton et modifier la couleur. Cet exercice m’a fait comprendre comment appliquer un comportement similaire à plusieurs éléments. Le rendu était réussi, et même si le design n’était pas exceptionnel, l’interactivité fonctionnait parfaitement.







Dans cet exercice, j’ai appris à manipuler la position d’un élément avec JavaScript en utilisant les propriétés left et top. Il fallait faire apparaître un carré de couleur à un endroit aléatoire au chargement, puis de le déplacer sous la souris à chaque clic. J’ai découvert comment récupérer les coordonnées de la souris avec event.clientX et event.clientY, et comment animer ce déplacement avec transition. La difficulté était assez faible, mais ça m’a aidé à comprendre comment faire interagir du code avec le comportement de l’utilisateur. Le résultat fonctionnait bien, même si le design restait vraiment hyper simple.






Ici, j’ai bossé sur un petit jeu où un papillon devait fuir le curseur quand on le survole. J’ai utilisé JavaScript pour déplacer l’image du papillon à un endroit aléatoire dans la prairie, grâce aux propriétés top et left. C’était l’occasion de revoir comment gérer les événements (mouseenter) et calculer des positions avec les tailles d’éléments. La difficulté était moyenne, surtout pour bien gérer les limites de l’écran. Le résultat était marrant et fonctionnait bien. C’était un exercice sympa pour lier interactivité et animation.
